Introducción

JavaScript es un lenguaje de programación multiplataforma orientado a objetos. Es un lenguaje pequeño y liviano. Dentro de un entorno anfitrión (por ejemplo, un navegador web), JavaScript se puede conectar al objetos de su entorno para proporcionar control programático sobre a ellos.

JavaScript contiene una biblioteca estándar de objetos, como Array, Fecha y Matemáticas, y un conjunto básico de elementos del lenguaje como operadores, estructuras de control y declaraciones. JavaScript central puede ampliarse para una variedad de propósitos completándolo con objetos adicionales; Por ejemplo:

  • JavaScript del lado del cliente amplía el lenguaje central proporcionando objetos para controlar un navegador y su modelo de objetos de documento (DOM). Por ejemplo, las extensiones del lado del cliente permiten que una aplicación coloque elementos en un formulario HTML y responder a eventos del usuario como el mouse clics, entrada de formularios y navegación de páginas.
  • JavaScript del lado del servidor amplía el lenguaje central proporcionando objetos relevantes para ejecutar JavaScript en un servidor. Por ejemplo, Las extensiones del lado del servidor permiten que una aplicación se comunique con un base de datos, proporciona continuidad de la información de una invocación a otra. otra de la aplicación, o realizar manipulaciones de archivos en una servidor.
JavaScript y Java

JavaScript y Java son similares en algunos aspectos pero fundamentalmente diferente en algunos otros. El lenguaje JavaScript se parece a Java pero no tiene el tipado estático ni la verificación de tipo fuerte de Java. JavaScript sigue la mayoría de las convenciones de nomenclatura y sintaxis de expresiones de Java. y construcciones básicas de flujo de control, que fue la razón por la que fue renombrado de LiveScript a JavaScript.

En contraste con el sistema de clases en tiempo de compilación de Java creado por declaraciones, JavaScript admite un sistema de ejecución basado en un pequeño número de tipos de datos que representan numéricos, booleanos y de cadena valores. JavaScript tiene un modelo de objetos basado en prototipos en lugar del modelo de objetos basado en clases más común. El modelo basado en prototipos. proporciona herencia dinámica; es decir, lo que se hereda puede variar para objetos individuales. JavaScript también admite funciones sin cualquier requisito declarativo especial. Las funciones pueden ser propiedades de objetos, ejecutándose como métodos poco tipificados.

JavaScript es un lenguaje de forma muy libre en comparación con Java. Tu no Tenemos que declarar todas las variables, clases y métodos. Usted no tiene preocuparse por si los métodos son públicos, privados o protegido y no es necesario implementar interfaces. variables, Los parámetros y los tipos de retorno de funciones no se escriben explícitamente.

Hola Mundo!

Utilice variables como nombres simbólicos para los valores en su aplicación. Los nombres de las variables, llamados identificadores, se ajustan a ciertas reglas.

Un identificador de JavaScript debe comenzar con una letra, un guión bajo (_) o signo de dólar ($); Los caracteres siguientes también pueden ser dígitos (0-9). Debido a que JavaScript distingue entre mayúsculas y minúsculas, las letras incluyen los caracteres "A" a "Z" (mayúsculas) y los caracteres "a" a "z" (minúscula).

Puede utilizar ISO 8859-1 o letras Unicode como å y ü en identificadores. También puede utilizar las secuencias de escape Unicode como caracteres en identificadores. Algunos ejemplos de nombres legales son Número_accesos, temp99 y _nombre.

Alcance de una variables

Cuando declaras una variable fuera de cualquier función, se llama variable global, porque está disponible para cualquier otro código en el documento actual. Cuando declaras una variable dentro de una función, se llama variable local, porque está disponible sólo dentro de ese función.

JavaScript anterior a ECMAScript 2015 no tiene declaración de bloqueo alcance; más bien, una variable declarada dentro de un bloque es local para el función (o alcance global) en la que reside el bloque. Para Por ejemplo, el siguiente código registrará 5, porque el alcance de x es el función (o contexto global) dentro de la cual se declara x, no el bloque, que en este caso es una declaración if.

if (true) { var x = 5; } console.log(x); // 5

Este comportamiento cambia cuando se utiliza la declaración let introducida en ECMAScript 2015.

if (true) { let y = 5; } console.log(y); // ReferenceError: y is not defined

if...else statement
Utilice la sentencia if para ejecutar una sentencia si se cumple una condición lógica. verdadero. Utilice la cláusula opcional else para ejecutar una declaración si el la condición es falsa. Una declaración if tiene el siguiente aspecto: if (condición) { declaración_1; } más { declaración_2; } La condición puede ser cualquier expresión que se evalúe como verdadera o falsa. Ver Booleano para una explicación de lo que se evalúa como verdadero y falso. Si la condición se evalúa como verdadera, se ejecuta la declaración_1; de lo contrario, Se ejecuta la instrucción_2. declaración_1 y declaración_2 pueden ser cualquiera declaración, incluidas más declaraciones if anidadas.

También puede componer las declaraciones usando else if para tener múltiples condiciones probadas en secuencia, de la siguiente manera:

if (condition_1) { statement_1; } else if (condition_2) { statement_2; } else if (condition_n) { statement_n; } else { statement_last; } En el caso de múltiples condiciones, solo la primera condición lógica que se evalúa como verdadero será ejecutado. Para ejecutar múltiples declaraciones, agrúpelas dentro de una declaración de bloque ({ ... }). En En general, es una buena práctica utilizar siempre declaraciones de bloque, especialmente al anidar declaraciones if: if (condition) { statement_1_runs_if_condition_is_true; statement_2_runs_if_condition_is_true; } else { statement_3_runs_if_condition_is_false; statement_4_runs_if_condition_is_false; } if (x = y) { /* declaraciones aquí */ } Si necesita usar una tarea en una expresión condicional, una práctica común es poner paréntesis adicionales alrededor de la tarea. Mientras
Una sentencia while ejecuta sus sentencias siempre que se cumpla un tiempo especificado. la condición se evalúa como verdadera. Una declaración while tiene el siguiente aspecto: while (condiction) statement Si la condición se vuelve falso, la declaración dentro del bucle deja de ejecutarse y el control pasa a la declaración que sigue al bucle.

La prueba de condición ocurre antes de que se ejecute la declaración en el bucle. Si la condición devuelve verdadera, se ejecuta la declaración y el La condición se prueba nuevamente. Si la condición devuelve falsa, ejecución se detiene y el control pasa a la declaración que sigue a while.

Para ejecutar varias declaraciones, utilice una declaración de bloque ({...}) para agrupa esas afirmaciones.

Ejemplo:

El siguiente bucle while se itera siempre que n sea menor que tres:

var n = 0; var x = 0; while (n < 3) { n++; x += n; }

Con cada iteración, el bucle incrementa n y agrega ese valor a x. Por tanto, x y n toman los siguientes valores:

  • Después del primer pase: n = 1 y x = 1
  • Después del segundo pase: n = 2 y x = 3
  • Después del tercero pase: n = 3 y x = 6

Después de completar el tercer pase, la condición n < 3 ya no es verdadero, por lo que el ciclo termina.

Funciones declaradas
Una definición de función (también llamada declaración de función o función declaración) consta de la palabra clave de función, seguida de:
  • El nombre de la funcion
  • Una lista de argumentos para la función, entre paréntesis y separado por comas.
  • Las declaraciones de JavaScript que definen la función, encerradas en llaves, { }.

Por ejemplo, el siguiente código define una función simple llamada cuadrado:

function square(number) { return number * number; }

La función cuadrado toma un argumento, llamado número. La función consta de una declaración que dice devolver el argumento del función (es decir, número) multiplicada por sí misma. El regreso La declaración especifica el valor devuelto por la función.

return number * number;

Los parámetros primitivos (como un número) se pasan a funciones mediante valor; el valor se pasa a la función, pero si la función cambia el valor del parámetro, este cambio no se refleja globalmente o en la función de llamada.
